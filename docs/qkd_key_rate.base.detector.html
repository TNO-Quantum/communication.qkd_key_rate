Parameters:

  • name (str) ‚Äì Label of the detector party

  • jitter_source (float) ‚Äì Time-delay introduced at the source

  • jitter_detector (float) ‚Äì Time-delay introduced at the detector

  • dead_time (float) ‚Äì The recovery time before a new event can be recorded

  • detection_window (float) ‚Äì Time window used for event detection

  • efficiency_system (float) ‚Äì Efficiency of the detecting side without the detector

  • polarization_drift (float) ‚Äì Shift/drift in the encoding of the photons

  • error_detector (float) ‚Äì Error-rate of detector

  • dark_count_frequency (Optional[float]) ‚Äì Number of dark-counts per second

  • dark_count_rate (Optional[float]) ‚Äì Number of dark-counts per detection window

  • detection_frequency (Optional[float]) ‚Äì Number of detection windows per second

  • interval (Optional[float]) ‚Äì Length of a single detection window in seconds

  • efficiency_detector (Optional[float]) ‚Äì Efficiency of the detector on the detecting side

  • efficiency_party (Optional[float]) ‚Äì Total efficiency of the detecting side

If related optional arguments are both given they must satisfy

  • \(\text{interval} = \frac{1}{\text{detection_frequency}}\)

  • \(\text{dark_count_frequency} = \text{dark_count_rate} \cdot \text{detection_frequency}\)

  • \(\text{efficiency_party} = \text{efficiency_system} \cdot \text{efficiency_detector}\)

If they are not given, they are calculated using the same formulas.

Returns:

Detector instance with specified parameters.

Raises:

  • ValueError ‚Äì If a required field is missing.

  • ValueError ‚Äì If a needed optional field is missing.

  • AssertionError ‚Äì If inconsistent related optional fields are provided.
